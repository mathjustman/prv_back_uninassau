# Prova Prática - Git e GitHub (Básico)

## Objetivo
Avaliar se o aluno consegue executar um fluxo básico de trabalho com Git/GitHub e responder perguntas conceituais.

## Regras da prova
Cada aluno deve realizar **todos** os passos abaixo:

1. Criar uma branch a partir da branch `develop`.
2. Criar um novo arquivo com as respostas das perguntas deste README.
3. Executar `git add` e `git commit` com uma mensagem clara.
4. Abrir um Pull Request para a branch `developer` (branch de destino da avaliação).

## Nome da branch
Use o padrão:

`prova/<seu-nome-sobrenome>`

Exemplo:

`prova/joao-silva`

## Arquivo de respostas
Crie um arquivo na raiz do projeto com o nome:



Exemplo:

`respostas-joao-silva.md`

No arquivo, responda as perguntas mantendo a numeração.

## Perguntas (Git e GitHub - básico)
1. Qual a diferença entre `git init` e `git clone`?
2. O que faz o comando `git status`?
3. Para que serve o `git add` antes do `git commit`?
4. Qual a diferença entre `git pull` e `git fetch`?
5. O que é um branch e por que ele é usado?
6. O que é um Pull Request no GitHub?
7. Explique a diferença entre branch de origem e branch de destino em um PR.
8. O que acontece se duas pessoas alterarem a mesma linha de um arquivo em branches diferentes?
9. Para que serve o arquivo `.gitignore`?
10. Qual é a função do `README.md` em um projeto?

## Fluxo esperado (resumo)
1. Atualizar branch local `develop`.
2. Criar branch da prova a partir de `develop`.
3. Criar e preencher o arquivo de respostas.
4. Rodar `git add` e `git commit`.
5. Enviar branch para o remoto.
6. Abrir PR para `developer`.

## Critérios de avaliação
A nota será baseada em:

- Execução correta do fluxo Git/GitHub.
- PR aberto corretamente para a branch `developer`.
- Clareza e correção das respostas.
- Organização (nome da branch, nome do arquivo e histórico de commit).

## Observação importante
Para evitar cópia entre alunos, a correção será feita individualmente pelos Pull Requests. Não compartilhe link do seu PR com colegas durante a prova.
